# 首次免责声明、应用内 Docs 与 README 重写设计

> 日期：2026-06-25  
> 分支：`feat/gui-and-packaging`  
> 范围：第一版打包发布后的体验完善

## 1. 背景与目标

第一版 GUI 已打包发布，下一步目标是完善首次使用体验与仓库门面：

1. 初次进入应用时必须弹出免责声明，用户同意后才能进入；设置页现有免责声明保留。
2. 顶部右上导航新增 `DOCS`，作为产品使用说明书；初次进入后引导用户点击。
3. 完全重写仓库 `README.md`，采用普通用户快速开始 + 开发者说明的双层结构。

本次设计采用轻量前端体验层方案：只修改前端静态资源与 README，不新增后端 API，不改数据库，不影响现有 CLI/GUI 打包入口。

## 2. 已确认决策

- Docs 形式：应用内页面，路由为 `#/docs`。
- 免责声明同意记录：按免责声明版本记录。当前版本同意后不再弹；未来文案版本升级可重新要求确认。
- README 结构：双层结构，顶部面向普通用户，后半面向开发者和维护者。
- 首次 Docs 引导：必须强调“最近一段时间不要用手机登录体适能 / 官方 APP”，并在 Docs 使用说明最前面重复强调。

## 3. 用户体验设计

### 3.1 免责声明启动门禁

应用启动后先检查本地同意状态。如果用户未同意当前免责声明版本，则展示阻断式免责声明弹窗/卡片。此时不渲染 Home、Accounts、Settings 等正常页面内容，用户必须先完成确认。

弹窗内容包含：

- `DISCLAIMER · 免责声明` 标题。
- 本程序仅用于学习和研究。
- 使用本程序辅助完成校园体测可能违反学校学生守则、课程纪律或平台规则。
- 由此产生的账号异常、课程处分、信用记录等后果由使用者本人承担。
- 作者及贡献者不对任何使用行为承担法律、纪律或行政责任。
- 账号凭证、跑步记录、运行日志均保存在本地，不主动上传到第三方。

操作按钮：

- 主按钮：`我已阅读并同意`。
- 次按钮：`暂不使用` 或 `退出/暂不使用`。

点击同意后写入本地状态并进入 Home。点击暂不使用时优先尝试关闭窗口；如果浏览器环境无法关闭，则提示用户直接关闭窗口。

### 3.2 首次 Docs 引导

用户首次同意免责声明并进入 Home 后，展示一次轻量引导弹窗/卡片。它不是普通欢迎提示，而是“使用前必读”的入口提醒。

引导中必须明确展示：

> 使用前请退出手机端体适能 / 官方 APP，并在最近一段时间内不要再次登录手机端，避免多端状态冲突或异常。

引导操作：

- 主按钮：`打开 DOCS`，点击后跳转 `#/docs`。
- 次按钮：`稍后再说`，点击后关闭引导。

无论用户点击主按钮还是稍后再说，都记录 `ts_docs_onboarding_seen`，避免每次启动反复打扰。未来如 Docs 引导内容有重大更新，可通过提升引导版本重新展示。

### 3.3 顶部 DOCS 入口

在 `frontend/index.html` 的顶部导航中新增：

```html
<a href="#/docs" data-nav>DOCS</a>
```

位置建议为：`HOME · ACCOUNTS · DOCS · SETTINGS`。这样首次引导指向明确，后续用户也能随时回到说明书。

### 3.4 应用内 Docs 页面

新增 `frontend/assets/pages/docs.js`，并在路由中注册 `#/docs`。页面采用现有 Operations Board 视觉语言，轻量文档布局即可，不引入复杂文档框架。

推荐内容结构：

1. **Before You Start / 使用前必读**
   - 使用前请退出手机端体适能 / 官方 APP。
   - 最近一段时间内不要再次登录手机端。
   - 操作期间不要多端同时在线。
   - 不同学校规则不同，使用风险由用户自行承担。
2. **Quick Start / 第一次使用**
   - 刷新学校列表。
   - 授权账号。
   - 如需要，先爬取历史路径或更新人脸。
   - 设置跑步类型和距离。
   - 开始任务并查看进度。
3. **Accounts / 账号**
   - 为什么需要先退出手机端账号。
   - 授权时需要的信息。
   - 本地保存说明。
4. **Running / 跑步**
   - 晨跑、阳光跑、自由跑的区别。
   - 距离填写规则。
   - 执行期间不要重复登录手机端。
5. **Paths / 路径**
   - 路径爬取用途。
   - 没有历史路径时的处理。
   - 复用路径的注意事项。
6. **Face / 人脸**
   - 更新人脸的用途。
   - 如果账号没有人脸图片会怎样。
   - 建议先在官方 APP 中上传。
7. **Distance / 里程**
   - 查询总里程的用途。
   - 失败时先检查账号授权。
8. **FAQ / 常见问题**
   - 启动失败。
   - WebView2 Runtime。
   - 学校刷新慢。
   - 登录失败。
   - 外链打不开。
9. **Disclaimer / 免责声明**
   - 与设置页和启动门禁保持同一份核心文案。

为减少重复维护，免责声明文案应抽成统一模块，供启动门禁、Settings、Docs 共同复用。

### 4.1 主要文件

  - 顶部导航新增 `DOCS`。
- `frontend/assets/app.js`
  - 注册 `#/docs` 路由。

  - 新增首次 Docs 引导逻辑。
- `frontend/assets/pages/docs.js`
  - 新增应用内产品说明书。
- `frontend/assets/pages/settings.js`
  - 保留现有免责声明区域，并改为复用统一文案。
- `frontend/assets/disclaimer.js`
  - 新增免责声明版本、标题、条目、前置提醒文案等常量。
- `frontend/assets/app.css`
  - 新增遮罩、弹窗、文档页面、引导卡片样式。
- `README.md`
  - 完全重写。

### 4.2 本地状态键

建议使用以下 localStorage 键：

- `ts_disclaimer_version`：用户已同意的免责声明版本，例如 `"1"`。
- `ts_disclaimer_accepted_at`：同意时间，ISO 字符串。
- `ts_docs_onboarding_seen`：是否已看过当前 Docs 引导。

可选：如果希望未来引导也支持版本升级，可使用 `ts_docs_onboarding_version` 替代布尔值。

### 4.3 启动流程

1. `DOMContentLoaded` 触发。
2. 如果 URL 中有 token，沿用现有逻辑写入 `sessionStorage`。
3. 设置主题。
4. 检查 `localStorage.ts_disclaimer_version` 是否等于当前免责声明版本。
5. 如果未同意：
   - 渲染免责声明 gate。
   - 不执行正常路由渲染。
6. 用户点击同意：
   - 写入 `ts_disclaimer_version` 与 `ts_disclaimer_accepted_at`。
   - 标记本次需要展示 Docs 引导。
   - 进入 `#/home` 并渲染。
7. Home 渲染完成后，如果 Docs 引导未展示过，则展示引导。
8. 用户点击 `打开 DOCS`：
   - 写入 Docs 引导已看状态。
   - 跳转 `#/docs`。
9. 用户点击 `稍后再说`：
   - 写入 Docs 引导已看状态。
   - 关闭引导。

## 5. 错误处理与降级

- `localStorage` 不可用：降级为本次会话内变量，用户仍能同意进入；刷新后可能再次出现免责声明。
- `window.close()` 失败：显示“请直接关闭窗口”的提示，不做危险或平台相关强制退出。
- `#/docs` 动态 import 失败：沿用现有路由错误卡片显示页面加载失败。
- 外链处理：沿用现有全局外链拦截逻辑，优先 `window.open`，失败时复制 URL 到剪贴板。
- 离线字体：保持现有字体 fallback，不因 Docs 页面新增依赖外部资源。
