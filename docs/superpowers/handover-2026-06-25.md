把现有命令行 `main.py`（体测能模拟跑步辅助工具）包装为面向**非技术学生**的桌面应用，并打包为单一 `TiShiNeng.exe`（Windows only，便携式）。

## 4. 与原计划的偏离

实施过程中发现的真实 bug，已修复并 commit，记在这里供评审：

### 4.1 SQLite `database is locked`（commit `6b087e5`）

**根因**：`schools/refresh` 原实现用同步 `httpx.get` 阻塞 async 事件循环，加上整 refresh 单事务 + flush（不 commit），SQLite 长事务下出错。
**修复**：改 `httpx.AsyncClient` 复用连接池；每个省份完成后 `await db.commit()`；单所学校/单省失败 skip 不让整 refresh 崩；响应多了 `skipped` 字段。

### 4.2 `schools/refresh` 改为异步任务 + WS 进度（commit `80680c1`）

**起因**：UI/UX 校验时用户反馈"刷新很慢没提示"。
**改动**：
- POST `/api/schools/refresh` 立即返回 `{task_id}`，后台 `asyncio.create_task` 跑 `_do_refresh`
- 每个省份开始处理前 emit `{phase: "refreshing", current, total, province, schools_added}`
- 完成后 emit `{phase: "done", total, skipped}`
- 起步前 `await asyncio.sleep(0.1)` 让 WS 来得及订阅（avoid 无缓冲 bus 丢首事件，和 Task 4.1 `emitting_run` 同一时序模式）
- `_do_refresh` 自己 `async for db in _raw_get_db():` 拿 session（path 不能用 `Depends(get_db)`）
- conftest 加 `wait_for_task_done` 公共 helper；5 个测试文件的 `_seed_*` 都用它等异步完成

### 4.3 uvicorn 不带 WebSocket 库（commit `a5778f0`）

**根因**：`requirements.txt` 装的是 minimal `uvicorn==0.32.0`，没带 `websockets` extras。测试能跑是因为 TestClient 内置 WS；实际 uvicorn serve 提示 `No supported WebSocket library detected`。
**修复**：requirements.txt 显式加 `websockets==13.1`。

### 4.4 `database.py` engine 在 import 时 bind（conftest 隐患）

**现象**：`tmp_data_dir` fixture 用 `monkeypatch.setenv("DATABASE_URL", ...)`，但 `database.py` 第 13 行的 `engine = create_async_engine(DATABASE_URL, ...)` 在模块 import 时就已经 bind，env 改不动它。
**Workaround**：conftest 改为 per-test `drop_all` + `create_all` 共享 engine（commit `461ed46` Phase 3.2 修复）。后果：测试运行时实际共用 `./tsn_data.db`（仓库根），GUI 跑着时跑测试会撞锁。
**风险**：测试与实际运行有耦合点，新模型若做更深改动需注意。完整修复需要 lazy engine 或者把 `database` 改为接受 URL 参数注入。

### 4.5 Phase 1 Task 1.4 — spiderServer 增加空 data 提前 break

**偏差**：plan 给的测试 mock `appSportRecordList.side_effect=[第一页, 空页]`，期望"空 data 立即 break"，但 plan 的 spiderServer 实现只在 `'data' not in resp` 才 break，对 `{"data": []}` 不会 break。
**实施者改动**：把 `if 'data' not in resp:` 扩展为 `if 'data' not in resp or not resp['data']:`。这是合理的潜在性能改进（少 9 次空请求），但属于 plan 外偏差。

### 4.6 UI/UX 重设计（commit `c173708`，覆盖 plan 的初始 CSS）

用户人工校验阶段执行 frontend-design 技能给的设计方向。**plan 里的 Phase 5/6 初始 CSS + 8 个页面文件被这一个 commit 几乎完全重写**。新方向：

- **主题**："TISHINENG OPERATIONS BOARD"（跑道调度台），与 plan 初始的"轻量 SaaS 蓝紫"截然不同
- **字体**：Anton (Google Fonts) + IBM Plex Sans Condensed + IBM Plex Mono — 通过 `<link rel="stylesheet">` 加载 Google CDN，**离线时会降级到系统字体**（fallback 已设置）
- **色板**：墨蓝 `#18222B` / 米纸 `#EFE9DC` / Solari 黄 `#C2872A` / 跑道红 `#A33D2A` / 苔绿 `#6E8050`
- **signature**：主页 6 张 LANE 卡片右上角 Anton 大字 01–06 + LANE 小标签
- **额外 UI 改动**：学校列表加搜索框、全部 emoji 已移除、设置页加 DISCLAIMER 卡片 + 两个仓库链接（FORK: `cd233ljx/TiShiNengRunning`，UPSTREAM: `dispose0335/TiShiNengRunning`，含说明文案）

### 5.2 已知的 Phase 9 阻塞问题（必须先解决再继续）

**问题**：plan 里建议的目录名 `packaging/` 与 PyPI 上 `packaging` 库冲突 — Python import `packaging` 时优先找到本地空目录（即使没有 `__init__.py` 也是 namespace package），导致 PyInstaller 自身的 import 失败。

**子代理中断前的最后一句**："There's a name conflict — our local `packaging/` directory shadows PyInstaller's required `packaging` library. Let me remove the `__init__.py` to avoid creating a regular package (it would still be a namespace package though, so this likely won't fix it)."

**建议解决**（任选其一）：

| 方案 | 改动 |
|---|---|
| A. 改目录名 | `packaging/` → `pkg_build/` 或 `tools/pyinstaller/`，更新 plan + scripts/verify-build.bat |
| B. 把 spec 放到项目根 | `tishineng.spec` 放仓库根，version_info 同；放弃 packaging 目录 |
| C. 改 spec 的 import 路径 | PyInstaller 启动时调 `sys.path` 把 `packaging/` 撤后，复杂且脆弱 |

**推荐 B**：spec 放根目录，无歧义，PyInstaller 通用做法。改动最小。
